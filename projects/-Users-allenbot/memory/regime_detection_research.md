# 市场 Regime Detection 研究（2026-02-26）

## 现有方案问题
- MA(7) vs MA(25) + RSI 太简单，没判断趋势强度
- cooldown_bars=3 在 15m 上太短（45分钟），导致 1680 次切换/年
- ETH 回测 166% vs proc 单独 312%，大部分币种 Regime 跑不过 proc

## 第一梯队：简单可靠

### ADX(14) — 趋势强弱（最关键的缺失拼图）
- ADX < 20: 无趋势 → ranging
- ADX 20-25: 灰色地带
- ADX > 25: 强趋势 → 再看方向
- 加密货币可能需要更高阈值（25-30）
- 延迟: ~14根K线（15m = 3.5小时）
- DI+ > DI- = 多头方向，DI- > DI+ = 空头方向

### SMA斜率 + ATR% — 方向+波动（实盘验证最佳）
- SMA(50) 最近5根斜率判方向
- ATR%(14) = ATR/Close vs 100日均值判波动
- 双重过滤，误报最少
- 有人用这个 NASDAQ 实盘 +100%
- 延迟: ~50根K线

### 布林带宽度 — 波动收缩/扩张
- BBW < 0.8x均值: 挤压（即将突破）— 唯一有预测性的信号
- BBW > 1.5x均值: 扩张
- 延迟: ~20根K线

## 第二梯队：高级但复杂

### HMM (隐马尔可夫)
- 概率框架，理论优雅
- QuantConnect实测: 对熊市反应太慢，容易过拟合
- 需要定期重训练，不推荐单独用

### Hurst 指数
- H > 0.5 趋势, H = 0.5 随机, H < 0.5 均值回归
- 需要 100-240 根K线，15m 上太慢（60小时窗口）
- 更适合 1h+ 周期

### GMM (高斯混合)
- 无监督聚类，零延迟
- 没有时间平滑，噪音太多

## 关键设计原则

### 滞后阈值（Hysteresis）比冷却更好
- 进入趋势: ADX > 25
- 退出趋势: ADX < 18
- 不同阈值进出，防止在边界反复横跳

### 最少停留时间
- MIN_REGIME_BARS = 12（15m = 3小时）

### Regime 当过滤器 > 当切换器
- QuantConnect 社区共识：不利条件下停止交易，比在策略间跳来跳去更稳健

## 建议改进方案（三层复合）
```
第一层: ADX(14) 判趋势强弱
  ADX < 20 → ranging → mean_reversion
  ADX > 25 → 有趋势，进第二层

第二层: SMA(50) 斜率判方向
  斜率 > 0 → bull → proc
  斜率 < 0 → bear → demandindex_short

第三层: ATR% vs 历史均值
  ATR% > 2倍均值 → 高波动，降低信心或暂停

+ 滞后阈值: 进趋势25/退趋势18
+ 最少停留: 12根K线
```

## 加密货币特殊考量
- 24/7 交易，周末流动性下降（价差宽11%，深度少9%）
- 基准波动率比股市高 3-5 倍，ADX 阈值要上调
- BTC 主导率上升+BTC 跌 = 避险；BTC 主导率下降 = 山寨季
- 日内 Hurst > 0.5 说明趋势跟踪在加密货币日内更有效
