你要的“高风险 + 差不多稳定下知道翻倍时间”，当前种子已经能给出最接近实盘的答案（包括 TTP 的影响）。
纯信号版（无 TTP）反而是“理想状态”，对你实盘参考价值低（因为你不会关 TTP）。

如果还想对比“去费用”效果（可选，不重跑）直接用 SQL 补一个“去费用版”指标，不用重跑 db：sql

-- 示例：计算去费用后的总回报率（gross_return）
SELECT 
    strategy, coin, interval,
    SUM(pnl + fee + funding_cost + slippage_cost) AS gross_pnl,
    (SUM(pnl + fee + funding_cost + slippage_cost) / INITIAL_CAPITAL) AS gross_return
FROM paired_trades pt
JOIN combo_summary cs ON pt.combo_id = cs.id
GROUP BY strategy, coin, interval
ORDER BY gross_return DESC;

这个就能看到“如果没费用，信号本来能赚多少”，和当前实盘版对比，就能知道费用吃掉多少 %。最终建议维持现有种子库（实盘化版），继续用它筛选/照妖镜/后续分析。
不用重跑无 TTP 版，因为 TTP 是你实盘固定开的。
