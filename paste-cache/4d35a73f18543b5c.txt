❯ 什么是14天涨跌幅？                                                            
                         
⏺ regime 标签的核心规则：                                                       
                                                                              
  开仓那一刻，往回看14天，计算这14天里价格涨了多少或跌了多少。                  
   
  - 涨超 +10% → 疯牛                                                            
  - 跌超 -10% → 疯熊                                                          
  - 中间 ± 10% + L1说熊 → 慢熊
  - 中间 ± 10% + 其他 → 慢牛或盘整

  人话：每笔交易开仓时，系统往后看两周的行情，用这两周的涨跌幅来判断"现在是什么
  市场"，然后贴标签。

  1m
  的问题就在这里——开一笔1m的单，拿14天的大趋势来判断它所处的市场，太粗了。就像你
  出门买瓶水，天气预报说这个月整体晴天，但此刻刚好在下雨，标签和实际完全对不上。

❯ 为什么不是分周期的呢？1m 有1m的定法15m有15m的定法才对呀                       
                                                         
⏺ 你说得对，这是一个真正的问题。                                                
                                                                              
  现在的做法是：所有周期统一用"14天涨跌幅"来判断市场状态。                      
   
  但14天对不同周期意义完全不同：                                                
  - 1h 交易：14天是合理的中期趋势窗口，匹配                                   
  - 1m 交易：1m的机会可能在几小时内出现，14天太粗了，早就错过了

  应该按周期匹配时间窗口：

  ┌──────┬──────────────┐
  │ 周期 │   建议窗口   │
  ├──────┼──────────────┤
  │ 1m   │ 1~2天        │
  ├──────┼──────────────┤
  │ 5m   │ 3~5天        │
  ├──────┼──────────────┤
  │ 15m  │ 7天          │
  ├──────┼──────────────┤
  │ 1h   │ 14天（当前） │
  └──────┴──────────────┘

  这样每个周期用"和自己量级匹配"的时间来判断市场状态，1m
  不用拿14天的大趋势来分班。
