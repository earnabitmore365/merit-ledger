  问题是：16 亿条记录的 balance_after 逐条用 Python + executemany UPDATE，每条都要查+写

  执事，balance_after 这步的方案有性能问题——16 亿条逐条 Python 累加 + UPDATE            
  可能需要很多小时。要不要 kill 掉换方案？                                              
                                                                                        
  替代方案：不用 UPDATE，用 DuckDB 的 CREATE TABLE AS SELECT + 窗口函数重建整个表（虽然 
  UPDATE 不支持窗口函数，但 CTAS 支持）：                         
                                                                                        
  CREATE TABLE paired_trades_new AS                                                     
  SELECT *, 500.0 + SUM(pnl_amount) OVER (                        
      PARTITION BY seed_type, combo_id                                                  
      ORDER BY open_candle_idx, close_candle_idx                                        
      ROWS UNBOUNDED PRECEDING                                    
  ) AS balance_after                                                                    
  FROM paired_trades;                                                                   
  DROP TABLE paired_trades;                                       
  ALTER TABLE paired_trades_new RENAME TO paired_trades;
