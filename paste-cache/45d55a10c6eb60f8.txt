好，仔细对比了两份方案。先说结论：大方向完全一致，差异在细节处理上。下面逐项对比：

  ---                                                                                 
  共识部分（两份方案一致）
                                                                                      
  - 只改 3 个文件（config.py, detector.py, strategy.py），0 新文件                  
  - backtest.py / system.py / trade_menu.py / indicators/ 全部不改
  - ADX 三层复合：ADX 判趋势 → SMA 判方向 → ATR% 防高波动
  - 滞后阈值：进趋势 ADX>25 / 退趋势 ADX<18
  - 最少停留 12 根 K 线
  - MarketScore 回测禁用（use_market_score=False）
  - MarketScore 不改变信号方向，只调 confidence
  - 验证标准：切换 <200次/年，5/10 币种胜出，Sharpe ≥ proc×80%

  ---
  关键差异

  1. 滞后阈值放哪里

  Column 1: detector
  我的方案: 直接用 adx_enter_threshold 判定 regime
  Sonnet 方案: 返回 naive_regime（用中间值 22 做粗判）
  ────────────────────────────────────────
  Column 1: strategy
  我的方案: 也有滞后逻辑（混合）
  Sonnet 方案: 独占滞后逻辑（_apply_hysteresis）

  Sonnet 更干净。 detector 保持纯函数/无状态，只负责"当前 ADX
  是多少"。滞后是有状态的（"我现在在趋势里还是 ranging 里"），放在 strategy
  更合理，因为 strategy 本来就有 _current_regime 状态。

  2. 高波动处理

  Column 1: 处理方式
  我的方案: 高波动 → regime 强制变 "ranging"
  Sonnet 方案: 高波动 → confidence × 0.5
  ────────────────────────────────────────
  Column 1: 效果
  我的方案: 直接换策略（从 proc 换成 mean_reversion）
  Sonnet 方案: 保留原策略，但降低信心

  Sonnet 更灵活。 高波动不一定意味着趋势结束，可能只是波动加大。直接换成
  mean_reversion 太激进了，降低 confidence 让 TradingSystem 自己判断更合理。

  3. ADX 计算方法

  ┌──────────┬───────────────────────────────────────┬────────────────────────┐
  │          │               我的方案                │      Sonnet 方案       │
  ├──────────┼───────────────────────────────────────┼────────────────────────┤
  │ 算法     │ 标准 Wilder 平滑（更精确）            │ 简单平均近似（更简单） │
  ├──────────┼───────────────────────────────────────┼────────────────────────┤
  │ 数据需求 │ min_klines=120（3×period + lookback） │ min_klines=60          │
  ├──────────┼───────────────────────────────────────┼────────────────────────┤
  │ 复杂度   │ ~80 行                                │ ~40 行                 │
  └──────────┴───────────────────────────────────────┴────────────────────────┘

  我的更精确，Sonnet 的更简洁。 但因为阈值带宽是 18-25（7
  个点的余量），简单平均的误差可能在可接受范围内。不过 Wilder 平滑是 ADX
  的标准定义，用标准方法更可靠。

  4. SMA 斜率计算

  ┌──────┬───────────────────────────┬──────────────────────────┐
  │      │         我的方案          │       Sonnet 方案        │
  ├──────┼───────────────────────────┼──────────────────────────┤
  │ 方法 │ 多个时点的 SMA 变化率平均 │ 前半段 vs 后半段均值之差 │
  ├──────┼───────────────────────────┼──────────────────────────┤
  │ 参数 │ sma_slope_window=5        │ 无额外参数               │
  └──────┴───────────────────────────┴──────────────────────────┘

  我的方法给出归一化的百分比斜率，Sonnet
  给出价格差值。对于"正还是负"的判断两个都够用，但我的多了一个可调参数。

  5. MarketScore 信心调整

  ┌──────┬─────────────────────────────┬─────────────────────────┐
  │      │          我的方案           │       Sonnet 方案       │
  ├──────┼─────────────────────────────┼─────────────────────────┤
  │ 基于 │ regime 方向：bull+高MS=加成 │ 信号方向：BUY+高MS=加成 │
  ├──────┼─────────────────────────────┼─────────────────────────┤
  │ 方式 │ 乘法（multiplier）          │ 加法（delta）           │
  ├──────┼─────────────────────────────┼─────────────────────────┤
  │ 公式 │ confidence × multiplier     │ confidence + delta      │
  └──────┴─────────────────────────────┴─────────────────────────┘

  Sonnet 更直观。 看信号方向比看 regime 更精确——有可能 regime 是 bull
  但子策略在某个时刻给出 HOLD。加法 (ms - 0.5) × 0.3 简单可控，最多
  ±0.15，不会有乘法的非线性问题。

  6. 缓存时间

  ┌──────────────────┬──────────┬─────────────┐
  │                  │ 我的方案 │ Sonnet 方案 │
  ├──────────────────┼──────────┼─────────────┤
  │ MarketScore 缓存 │ 10 分钟  │ 5 分钟      │
  └──────────────────┴──────────┴─────────────┘

  差别不大，5 分钟更贴合 HyperLiquid 的 5 分钟缓存 TTL。
