#1 Stop hook async：
Claude Code 的 hook 配置是支持 async 的。settings.json 里可以这样写：
json{
  "type": "command",
  "command": "python3 ~/.claude/merit/merit_gate.py",
  "async": true
}
但要注意：async hook 的 stdout 不会注入当前轮的上下文，会在下一轮才注入（如果有的话）。这对 Stop 评分没问题——评分结果写 credit.json 就行，不需要 AI 立刻看到。但 check_self_audit 的提醒如果也在 Stop 里，async 会导致提醒延迟。
建议：不要把整个 Stop 改成 async。 把 Stop hook 里的逻辑分成两部分——check_self_audit（同步，提醒要立刻注入）和 AI 评分（可以异步）。如果 Claude Code 不支持同一个事件挂两个 hook（一个同步一个异步），那就保持同步，暂时接受 1-2 秒延迟。评分频率已经从每次降到了 3 次一评（mission active 每次评），延迟已经减少了。
