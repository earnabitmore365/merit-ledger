# 子类详细规则

（/reflect 升级的规则会追加到这里，每条包含：用户原话、出现次数、具体规则、案例）

---

## 如何填写

1. **跟你的 AI 讨论**你希望它遵守什么规则
2. **用 /reflect** 让 AI 从对话中自动提炼规则
3. AI 提案后你**批准**，规则自动写入
4. **父类**（INJECT 区域）由 hook 每次注入，AI 必定看到
5. **子类**（INJECT 外）是详细说明，AI 遇到具体场景时查阅

### 规则格式建议

```
## RUL-001｜规则标题

- **出现次数**：N次
- **用户原话**："..." （保留原文，帮助 AI 理解语气和严重程度）
- **规则**：具体要求
- **案例**：具体场景说明
```

### 父类 vs 子类

- **父类**：写在 `<!-- INJECT START -->` 和 `<!-- INJECT END -->` 之间，每条一行，精简
- **子类**：写在 INJECT END 之后，详细展开，含原话和案例
- **精炼原则**：子类变了 → 检查父类是否还覆盖 → 不覆盖就改父类
